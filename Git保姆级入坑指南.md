# Git 保姆级日常使用教程

如果你是第一次使用 Git，不要慌！这篇教程不仅告诉你“输入什么命令”，还告诉你“在**哪里**输入”以及“这些奇怪的字母到底**是什么意思**”。

---

## 📍 零、我在哪里输入这些命令？

要执行 Git 命令，你需要打开一个**命令行终端（Terminal）**。对于现在这个 `deepseek-app-reborn` 项目，你有两种最常用的方法打开它：

### 方法 1：在 VS Code 编辑器里打开（🌟 最推荐）
1. 用 VS Code 把 `deepseek-app-reborn` 这个文件夹整个打开。
2. 在顶部菜单栏点击 **终端 (Terminal)** -> **新建终端 (New Terminal)**。
3. 软件底部会弹出一个黑黑的框，这里面就是你输入代码、打命令的地方了！

### 方法 2：在 Windows 文件夹里直接打开
1. 打开 `deepseek-app-reborn` 所在的普通文件夹窗口。
2. 在这个窗口**最上方显示路径的地址栏**（比如写着 `F:\...\deepseek-app-reborn` 的地方），用鼠标点一下。
3. 把路径全删掉，直接输入 `cmd` 这三个字母，然后按回车。
4. 此时会弹出一个黑色的命令行窗口，你就可以在里面打字了。

## 🛠️ 一、日常三步走：把写好的代码推送到 GitHub (Push)

当你在电脑上写完了一天代码，或者删除了某些没用的文件，想把这些改动保存到大网（GitHub）上时，永远是这雷打不动的 **三步组合拳**：

### 第 1 步：选中全部改动 (Add)
```bash
git add .
```
- **`git`**: 呼叫 Git 软件来干活。
- **`add`**: 命令它把文件“添加到暂存区（准备提交的购物车）”。
- **`.` (那个点)**: 代表“当前文件夹下的**所有**东西”。也就是说：“把我今天在这个项目里新建的、修改的、甚至是删除的所有文件，全都扔进购物车里！”

### 第 2 步：打包并写备注 (Commit)
```bash
git commit -m "今天新增了登录功能，并且删除了测试用的无用文件"
```
- **`commit`**: 意思是“确认提交”，把刚才购物车里的东西正式打包成一个**时光机里的版本记录**。
- **`-m`**: 代表 **message (信息/留言)**。这是一个参数，告诉 Git：“我接下来双引号 `""` 里写的字，就是给这次打包写的备注留言”。
- 如果你不加 `-m`，Git 会不知所措，弹出一个非常难操作的黑色文本框逼你写一句话。所以加上 `-m "你的说明文字"` 是最方便的写法！

### 第 3 步：发射到宇宙 (Push)
```bash
git push origin main
```
- **`push`**: 意思是“推上去”，往互联网上发。
- **`origin`**: 是 Git 给远程 GitHub 网址起的**默认外号（代号）**。代表“原始的那个网上仓库地址”。
- **`main`**: 是你当前主要所在的 **分支 (Branch) 名字**（以前老系统叫 master，现在都叫 main 主分支）。
- 连起来的意思就是：“把我刚才打包好的版本，推送到 `origin(网上)` 的 `main(主线)` 上去！”

> **💡 如果你在本地删除了文件怎么传？**
> 也是一模一样的这三步：`git add .` -> `git commit -m "删除了某文件"` -> `git push origin main`。Git 没你想的那么笨，当你 `add .` 时，它就会自动发现某些文件不见了，并在打包时自动把删除操作记录下来推上网。

## 📥 二、写代码前要做的事：拉取最新代码 (Pull)

如果你的项目被别人改了，或者你换了一台电脑（比如从公司换到了家里的电脑），你在继续写代码**之前**，一定要先把最新进度下载下来：

```bash
git pull origin main
```
- **`pull`**: 意思是“拉下来”。获取大网上的最新变动，并和本机的代码合并。
- 这条命令就像是给你的本地电脑点击了“同步刷新进度”。

## 🛡️ 三、隐身披风：怎么指定文件不上传？ (.gitignore)

如果你电脑里有一些**绝对不能暴露**的东西（比如包含私人 API 密码的 `config.json`，或是几百 MB 大小的生成文件夹 `node_modules`），你要告诉 Git 假装看不见它们。

**操作方法：**
1. 在项目文件夹的最外面新建一个名字叫 **`.gitignore`** 的纯文本文件（注意最前面有个点）。
2. 在里面写上你不想上传的文件名或文件夹名。例如：
   ```text
   # 这是注释
   backend/config.json
   frontend/node_modules/
   *.log
   ```
   *(注：`*.log` 代表忽略所有以 .log 结尾的文件)*
3. 只要文件名字写在了这个清单里，你以后在命令行敲 `git add .` （把全部加进购物车）时，Git 就**绝对不会**把它们加进去。

## ⏪ 四、时光倒流：我写错代码了该怎么撤销？

有的时候你瞎改了一通把项目搞崩了，想要恢复成最近一次刚从网上拉下来的那个纯净状态，你可以清空所有的未保存修改：

**（⚠️ 极度危险：这会让你今天写的还没提交的代码全部灰飞烟灭！）**
```bash
git restore .
```
- **`restore`**: 意思是“恢复/还原”。把当前文件夹里的所有文件，强行倒退回最近一次 Commit 的状态。 
- 在你走投无路需要“重新读档”时有奇效。
